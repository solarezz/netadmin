# Настройка маршрутизации для доступа к WebFig (MikroTik) с Windows ПК

Эта инструкция нужна для того, чтобы ваш основной ПК (Windows) имел прямой IP-доступ к устройствам во внутренней сети лаборатории (например, к MikroTik на `10.1.1.2`), доступ к которым есть только у сервера SRV1.

Docker по умолчанию блокирует транзитный трафик (FORWARD), поэтому попытки прямого доступа (даже при наличии маршрутов) отвергаются. Решается это настройкой NAT и Forward-правил на сервере-шлюзе (SRV1).

## Шаг 1. Настройка шлюза (SRV1 - Linux)
Выполните эти команды в терминале SRV1 (через SSH):

**1. Включаем форвардинг пакетов на уровне ядра Linux:**
```bash
sudo sysctl -w net.ipv4.ip_forward=1
```

**2. Разрешаем транзитный трафик в файрволе:**
(По умолчанию Docker ставит политику DROP на цепочку FORWARD)
```bash
sudo iptables -A FORWARD -j ACCEPT
```

**3. Включаем NAT (Masquerade):**
Это нужно, чтобы MikroTik думал, что запросы приходят напрямую от SRV1, и не пытался отправить ответы в неизвестную ему подсеть 192.168.56.x.
```bash
sudo iptables -t nat -A POSTROUTING -s 192.168.56.0/24 -j MASQUERADE
```

---

## Шаг 2. Настройка маршрута (Ваш основной ПК - Windows)
Чтобы Windows знал, что искать устройства `10.1.x.x` нужно через сервер SRV1 (`192.168.56.101`), нужно прописать маршрут.

Запустите **PowerShell** или **Командную строку** от имени **Администратора** и выполните:
```cmd
route add 10.1.1.0 mask 255.255.255.0 192.168.56.101
```

*(Если оборудования в сети 10.1.x.x больше, можно прописать маску 255.255.0.0).*

---

## Шаг 3. Включение веб-интерфейса на самом MikroTik
По умолчанию MikroTik отключает порт 80 (www) для безопасности. Включить его можно через NetAdmin "Выполнить команду" или SSH:
```bash
/ip service set www disabled=no
```

✅ **Готово!** Теперь кнопки WebFig в `NetAdmin` будут мгновенно открывать интерфейс роутеров прямо в вашем браузере.
